# Автокликер для игры "Word City: Connect Word Game" (Android)

Этот проект представляет собой автоматический бот-кликер на Python для прохождения английской версии игры **Word City: Connect Word Game** на реальном Android-устройстве или эмуляторе.

## Стек технологий
*   **Управление устройством:** `uiautomator2` — для быстрого выполнения сложных жестов (свайпов по буквам без отрыва пальца).
*   **Компьютерное зрение:** `opencv-python` — для поиска контуров букв и шаблонного совпадения (template matching) кнопок рекламы/меню.
*   **Распознавание текста:** `easyocr` — для распознавания символов букв.
*   **Логика словаря:** автоматическое скачивание и фильтрация ~10,000 популярных английских слов с автоматическим решением анаграмм.

## 1. Подготовка Android-устройства

### На физическом телефоне:
1.  Перейдите в **Настройки** -> **О телефоне** и нажмите на **Номер сборки** 7 раз, чтобы включить режим разработчика.
2.  Перейдите в **Настройки** -> **Система** (или **Дополнительно**) -> **Для разработчиков**.
3.  Включите **Отладку по USB** (USB Debugging).
4.  *(Для некоторых устройств, например Xiaomi/Redmi)* Включите **Отладку по USB (Настройки безопасности)** (USB debugging (Security settings)) — это нужно для разрешения симуляции кликов и жестов через ADB.
5.  Подключите телефон к компьютеру по USB-кабелю. На экране телефона появится запрос на разрешение отладки с этого компьютера — выберите **Разрешить всегда**.

### На эмуляторе (LDPlayer, Nox, BlueStacks):
1.  В настройках эмулятора включите **ADB** (или USB Debugging).
2.  Обычно эмуляторы уже предоставляют права разработчика по умолчанию.

## 2. Установка зависимостей

У вас должен быть установлен Python версии 3.8 или выше. Откройте командную строку Windows (`cmd.exe`) или PowerShell и выполните установку библиотек:

```powershell
pip install uiautomator2 opencv-python easyocr numpy pillow
```

*Примечание: При первом запуске `easyocr` автоматически скачает веса нейросетевых моделей для распознавания английского языка (~15-30 МБ).*

## 3. Подключение и калибровка

1.  Убедитесь, что компьютер видит ваше устройство. Введите команду в командной строке или PowerShell:
    ```cmd
    adb devices
    ```
    Должно отобразиться ваше устройство со статусом `device`.
    *(Если командная строка пишет, что adb не найден, убедитесь, что путь к ADB добавлен в системную переменную PATH или запускайте команду `./adb devices` из папки с установленным ADB).*
2.  Откройте игру **Word City** на телефоне и зайдите на уровень, где виден круг с буквами.
3.  Запустите скрипт калибровки:
    ```cmd
    python calibrate.py
    ```
4.  Скрипт сделает скриншот и сохранит файлы в папку `debug_screenshots/`:
    *   `full_screen.png` — полный снимок экрана.
    *   `debug_roi_placement.png` — полный снимок с наложенной желтой рамкой круга букв.
    *   `debug_threshold.png` — бинаризованная маска круга букв (черно-белая).
    *   `debug_calibration.png` — область круга, где зеленым цветом обведены распознанные контуры букв, а красными крестиками — их центры.
5.  **Настройка координат:**
    Откройте `debug_roi_placement.png`. Рамка `CIRCLE ROI AREA` должна ровно очерчивать круг с буквами.
    Если рамка смещена или круг не попадает в нее, откройте файл [config.py](file:///d:/Projects/Autoclicker/config.py) и скорректируйте координаты `CIRCLE_ROI`:
    *   `x`, `y` — координаты левого верхнего угла рамки.
    *   `w`, `h` — ширина и высота рамки.
    Снова запустите `python calibrate.py` и убедитесь, что все буквы обведены зелеными контурами, а их центры определены верно.

## 4. Создание шаблонов кнопок (Критически важно)

Для того чтобы бот понимал, когда нажимать кнопку «Далее» (Next/Continue) для перехода на новый уровень, и как закрывать рекламу, вам нужно добавить шаблоны кнопок в папку `templates/`.

1.  Откройте сделанный ранее полный скриншот `debug_screenshots/full_screen.png` в любом графическом редакторе (Paint, Photoshop и т.д.).
2.  **Шаблон для перехода на следующий уровень:**
    *   Когда вы побеждаете на уровне, сделайте скриншот.
    *   Аккуратно вырежьте кнопку **Next** (или **Continue**) по контуру.
    *   Сохраните вырезанный фрагмент в папку `templates/` под именем `next.png` или `next_level.png` (формат PNG обязателен).
3.  **Шаблоны рекламных крестиков:**
    *   При появлении рекламы сохраните скриншот экрана (с помощью калибровочного скрипта или вручную).
    *   Вырежьте кнопку закрытия рекламы (обычно это крестик `[X]` в углах экрана).
    *   Сохраните в папку `templates/` с именем, начинающимся на `close` или `x`, например: `close_ad.png`, `x_button.png`, `close_1.png`.
    *   *Совет: Сделайте 2-3 шаблона для разных видов крестиков, так как рекламодатели используют разные стили кнопок закрытия.*

## 5. Запуск автокликера

После того как вы настроили `config.py` и добавили шаблоны в `templates/`, откройте игру на телефоне и запустите бота из консоли:

```cmd
python bot.py
```

### Логика работы бота:
1.  **Распознавание:** бот сканирует область круга, с помощью OpenCV находит координаты букв, вырезает каждую букву и через EasyOCR распознает символ.
2.  **Поиск слов:** генерируются все возможные слова на английском языке из найденных букв (длиной от 3 до N букв), фильтруются по словарю `words.txt` и сортируются по длине (сначала вводятся длинные слова).
3.  **Ввод свайпами:** с помощью функции `d.swipe_points` бот делает плавные свайпы между координатами букв.
4.  **Watchdog (Реклама и сбои):** параллельно работает фоновый процесс, который раз в 1.5 секунды проверяет имя активного приложения. Если открылся Play Market или браузер, бот нажимает кнопку `Back` на телефоне, возвращаясь в игру. Также он сканирует экран на наличие шаблонов крестиков закрытия рекламы и нажимает на них.
5.  **Победа:** при обнаружении кнопки перехода на следующий уровень (из папки `templates/`), бот автоматически кликает по ней и ждет загрузки следующей сцены.

## Настройка конфигурации

Основные настройки в файле `config.py`:
*   `DEVICE_SERIAL` — серийный номер вашего устройства (если их подключено несколько).
*   `CIRCLE_ROI` — область захвата круга букв.
*   `BINARY_THRESHOLD` — порог бинаризации. Если буквы плохо определяются зеленым контуром в `debug_calibration.png` (сливаются с фоном или разрываются), попробуйте уменьшить или увеличить это значение (обычно от 100 до 160).
*   `MIN_LETTER_AREA` / `MAX_LETTER_AREA` — фильтр размера контуров букв.
*   `TEMPLATE_THRESHOLD` — чувствительность поиска шаблонов (по умолчанию `0.75`). Если бот ложно кликает или не видит кнопку, измените этот параметр.
*   `WATCHDOG_INTERVAL` — частота проверки рекламы фоновым процессом.
*   `AD_SEARCH_REGION_RATIO` — доля экрана для поиска рекламных крестиков (по умолчанию `0.25` = верхние 25%).
*   `AD_COOLDOWN_SECONDS` — пауза OCR после обнаружения рекламы (по умолчанию `3.0` секунды).
*   `SHOW_TAPS` — включить отображение нажатий на экране устройства (`True` / `False`).

## AI Self-Healing (OpenRouter)

Бот поддерживает автоматическое восстановление через AI, когда все штатные стратегии исчерпаны.

### Эскалационная цепочка:
1. **Веб-ответы по номеру уровня** (2 попытки) — скачивание с wordcityanswers.com
2. **Веб-поиск по буквам** (2 попытки) — поиск уровня по распознанным буквам
3. **Локальный словарь** (2 попытки) — анаграммы из Scrabble-словаря
4. **AI Self-Healing** — vision-модель распознаёт буквы и генерирует слова

### Настройка:
1. Получите API ключ на [openrouter.ai/keys](https://openrouter.ai/keys)
2. Скопируйте `.env.example` в `.env`:
   ```cmd
   copy .env.example .env
   ```
3. Впишите ваш API ключ в файл `.env`:
   ```
   OPENROUTER_API_KEY=sk-or-v1-ваш-ключ
   ```
4. Бот автоматически активирует AI Self-Healing при наличии ключа.

### Параметры AI в `config.py`:
*   `AI_HEALING_ENABLED` — включить/выключить AI (по умолчанию `True`).
*   `AI_HEALING_MODEL` — модель OpenRouter (по умолчанию `nvidia/nemotron-3-nano-omni-30b-a3b-reasoning:free`).
*   `AI_HEALING_COOLDOWN` — минимальный интервал между AI-запросами (по умолчанию `10` секунд).
*   `STUCK_AI_HEALING_LIMIT` — через сколько неудач вызывать AI (по умолчанию `6`).

## Отладка (Show Taps)

Для визуализации нажатий бота на экране устройства:
1. Установите `SHOW_TAPS = True` в `config.py` (включено по умолчанию).
2. При запуске бот автоматически включит **Developer Options → Show taps** на устройстве.
3. При остановке бота (Ctrl+C) отображение нажатий автоматически выключится.

Также можно включить вручную: **Настройки → Для разработчиков → Показывать нажатия**.
